# ACP2E-4786: l’esportazione del prodotto non riesce quando è configurato lo storage remoto AWS S3

La patch ACP2E-4786 risolve il problema se l’esportazione del prodotto non riesce perché il driver di archiviazione utilizza un percorso non valido quando l’archiviazione remota AWS S3 è abilitata. Questa patch è disponibile quando è installato [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81. L’ID della patch è ACP2E-4786. Questo problema è pianificato per la risoluzione in Adobe Commerce 2.4.9.

## Prodotti e versioni interessati

**La patch è stata creata per la versione di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.7-p4

**Compatibile con le versioni di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.5 - 2.4.9

>[!NOTE]
>
>La patch potrebbe diventare applicabile ad altre versioni con le nuove versioni di [!DNL Quality Patches Tool]. Per verificare se la patch è compatibile con la versione di Adobe Commerce in uso, aggiornare il pacchetto `magento/quality-patches` alla versione più recente e verificare la compatibilità nella pagina [[!DNL Quality Patches Tool]: Cerca patch](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilizza l’ID patch come parola chiave di ricerca per individuare la patch.

## Problema

L’esportazione del prodotto non riesce se l’archiviazione remota AWS S3 è abilitata a causa di una gestione del percorso non valida nel driver di archiviazione.

<u>Passaggi da riprodurre</u>:

1. Configura [!DNL AWS S3] come archivio remoto:

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Disattivare l&#39;archiviazione del database per i file multimediali eseguendo il comando seguente:

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. Nella barra laterale di amministrazione, vai a **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Esporta**.
1. Per **[!UICONTROL Entity Type]**, scegliere **[!UICONTROL Products]**.
1. Fare clic su **[!UICONTROL Continue]**.

<u>Risultati previsti</u>:

Il file di esportazione è stato creato.

<u>Risultati effettivi</u>:

1. Il file di esportazione non è stato creato.
1. I registri mostrano un errore *SignatureDoesNotMatch*.
