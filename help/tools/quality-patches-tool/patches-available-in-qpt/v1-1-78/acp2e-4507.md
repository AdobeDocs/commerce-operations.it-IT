# ACP2E-4507: la configurazione di [!UICONTROL Password Options] non è applicabile alle richieste di reimpostazione della password del cliente effettuate tramite mutazioni GraphQL

La patch ACP2E-4507 risolve il problema per cui la configurazione **[!UICONTROL Password Options]** non viene applicata per le richieste di reimpostazione della password del cliente effettuate tramite mutazioni GraphQL. Questa patch è disponibile quando è installato [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. L’ID della patch è ACP2E-4507. Questo problema è pianificato per la risoluzione in Adobe Commerce 2.4.9.

## Prodotti e versioni interessati

**La patch è stata creata per la versione di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.8-p3

**Compatibile con le versioni di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>La patch potrebbe diventare applicabile ad altre versioni con le nuove versioni di [!DNL Quality Patches Tool]. Per verificare se la patch è compatibile con la versione di Adobe Commerce in uso, aggiornare il pacchetto `magento/quality-patches` alla versione più recente e verificare la compatibilità nella pagina [[!DNL Quality Patches Tool]: Cerca patch](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilizza l’ID patch come parola chiave di ricerca per individuare la patch.

## Problema

La configurazione **[!UICONTROL Password Options]** non è applicata per le richieste di reimpostazione della password del cliente effettuate tramite mutazioni GraphQL.

<u>Passaggi da riprodurre</u>:

1. Controlla e conferma le opzioni della password cliente in
   **[!UICONTROL Store]** > **[!UICONTROL Configurations]** > **[!UICONTROL Customers]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Password Options]**.
1. Inviare un messaggio e-mail di reimpostazione della password utilizzando la mutazione GraphQL `requestPasswordResetEmail`.
1. Reimpostare la password utilizzando la mutazione GraphQL `resetPassword` utilizzando il token dell&#39;e-mail inviata.
1. Ripetere gli stessi passaggi per reimpostare la password più volte.

<u>Risultati previsti</u>:

Il ripristino della password del cliente attivato tramite GraphQL si comporta come il ripristino della password basata sul Web e rispetta le impostazioni di configurazione [!UICONTROL Admin].

<u>Risultati effettivi</u>:

Le richieste di reimpostazione della password effettuate tramite GraphQL possono essere attivate un numero illimitato di volte e non applicano i limiti definiti nella configurazione della password del cliente [!UICONTROL Admin].
