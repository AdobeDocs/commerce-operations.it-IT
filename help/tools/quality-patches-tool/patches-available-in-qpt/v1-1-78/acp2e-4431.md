# ACP2E-4431: [!UICONTROL Related Products] corrispondenti alle regole di destinazione vengono eliminati durante il processo di reindicizzazione

La patch ACP2E-4431 risolve il problema per cui **[!UICONTROL Related Products]** corrispondente alle regole di destinazione viene eliminato durante il processo di reindicizzazione. Questa patch è disponibile quando è installato [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. L’ID della patch è ACP2E-4431. Questo problema è pianificato per la risoluzione in Adobe Commerce 2.4.9.

## Prodotti e versioni interessati

**La patch è stata creata per la versione di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.7-p4

**Compatibile con le versioni di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>La patch potrebbe diventare applicabile ad altre versioni con le nuove versioni di [!DNL Quality Patches Tool]. Per verificare se la patch è compatibile con la versione di Adobe Commerce in uso, aggiornare il pacchetto `magento/quality-patches` alla versione più recente e verificare la compatibilità nella pagina [[!DNL Quality Patches Tool]: Cerca patch](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilizza l’ID patch come parola chiave di ricerca per individuare la patch.

## Problema

I **[!UICONTROL Related Products]** corrispondenti alle regole di destinazione vengono eliminati durante il processo di reindicizzazione.

<u>Passaggi da riprodurre</u>:

1. Crea 10000 o più prodotti.
1. Assegna 10000 o più prodotti a 5000 o più categorie.
1. Crea 100 o più regole prodotto correlate.
1. Configura le condizioni della regola in cui **[!UICONTROL Products]** corrisponde all&#39;uso **[!UICONTROL Product Category]** è *X*.
1. Configurare **[!UICONTROL Products]** per la visualizzazione con **[!UICONTROL Product Category]**: **[!UICONTROL Constant Value]** *Y*.
1. Eseguire questa query GraphQL per recuperare `related_products` per un prodotto:

   ```graphql
   {
       products(filter: { url_key: { eq: "simple-product-491" } }) {
           items {
               sku
               related_products {
                   uid
                   sku
                   name
                   url_key
                   stock_status
                   price_range {
                       minimum_price {
                           regular_price {
                              value
                           }
                           final_price {
                               value
                           }
                       }
                   }
               }
           }
       }
   }
   ```

1. Creare un backlog di grandi dimensioni per l&#39;indice `targetrule_product_rule`.
1. Creare un backlog di grandi dimensioni per l&#39;indice `targetrule_rule_product`.
1. Esegui l&#39;indicizzatore `targetrule_rule_product`.
1. Esegui l&#39;indicizzatore `targetrule_product_rule`.

<u>Risultati previsti</u>:

L&#39;elenco `related_products` è sempre compilato con prodotti che soddisfano le condizioni della regola.

<u>Risultati effettivi</u>:

L&#39;elenco `related_products` restituisce un array vuoto durante l&#39;esecuzione degli indicizzatori.
