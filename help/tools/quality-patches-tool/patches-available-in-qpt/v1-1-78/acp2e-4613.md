# ACP2E-4613: una directory multimediale di grandi dimensioni causa ritardi nel caricamento della struttura di directory di Media Gallery

La patch ACP2E-4613 risolve il problema relativo alle strutture di directory multimediali di grandi dimensioni che causano risposte lente di `gettree`, con tempi di caricamento estesi per la struttura di directory **[!UICONTROL Media Gallery]**. Questa patch è disponibile quando è installato [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. L’ID della patch è ACP2E-4613. Questo problema è pianificato per la risoluzione in Adobe Commerce 2.4.9.

## Prodotti e versioni interessati

**La patch è stata creata per la versione di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.8-p3

**Compatibile con le versioni di Adobe Commerce:**

* Adobe Commerce (tutti i metodi di implementazione) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>La patch potrebbe diventare applicabile ad altre versioni con le nuove versioni di [!DNL Quality Patches Tool]. Per verificare se la patch è compatibile con la versione di Adobe Commerce in uso, aggiornare il pacchetto `magento/quality-patches` alla versione più recente e verificare la compatibilità nella pagina [[!DNL Quality Patches Tool]: Cerca patch](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=it). Utilizza l’ID patch come parola chiave di ricerca per individuare la patch.

## Problema

Le strutture di directory multimediali di grandi dimensioni causano risposte `gettree` lente, con tempi di caricamento estesi per la struttura di directory [!UICONTROL Media Gallery].

<u>Prerequisiti</u>:

1. Esiste un numero elevato di prodotti con più immagini.

<u>Passaggi da riprodurre</u>:

1. Genera miniature immagine per aumentare l&#39;ingombro dei file multimediali:

   ```
   bin/magento catalog:images:resize
   ```

1. Creare più sottodirectory in `pub/media/catalog/category` (ad esempio: test, test2, test3, test4 e cartelle nidificate aggiuntive in base alle esigenze).
1. Copiare un set di file multimediali a catalogo di grandi dimensioni nelle nuove directory create per aumentare in modo significativo le dimensioni della struttura delle directory multimediali, ad esempio:

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. Accedi al pannello di amministrazione e passa a **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Il completamento della richiesta `admin/media_gallery/directories/gettree/?isAjax=true` richiede molto tempo.

<u>Risultati previsti</u>:

La struttura della directory viene caricata in pochi secondi.

<u>Risultati effettivi</u>:

Il caricamento della struttura di directory richiede diversi minuti, a seconda del numero di sottodirectory in `pub/media`.
